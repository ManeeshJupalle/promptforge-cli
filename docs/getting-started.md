# Getting started

Ninety seconds from install to your first passing test.

## 1. Install

```bash
npm install -g promptforge
```

Requires Node 20 or later.

> **ESM-only.** `promptforge.config.ts` and `*.test.ts` files are loaded
> through Node's ESM resolver. If your project's `package.json` doesn't
> declare `"type": "module"`, add it (or stick to `*.test.yaml`). CommonJS
> projects will hit `Cannot require() ES Module ... in a cycle` when the
> TypeScript loader runs.

> **Global vs local install.** The CLI runs fine as a global install, but
> if you want TypeScript autocomplete inside `promptforge.config.ts` or
> `*.test.ts` files â€” or if your code imports `defineConfig` / `defineTestSuite`
> helpers â€” add the package to your project too:
>
> ```bash
> npm install --save-dev promptforge
> ```
>
> Without the local install, `import { defineConfig } from 'promptforge'` in
> your config file will fail with `Cannot find package 'promptforge'`. The
> scaffold generated by `promptforge init` avoids this import on purpose so
> the out-of-the-box flow works with just the global CLI.

## 2. Scaffold

```bash
cd your-llm-project
promptforge init
```

The interactive scaffolder asks four questions: test directory, which
providers to enable by default, whether to create an example test, and
whether to add `.promptforge/` to `.gitignore`.

It writes:

- `promptforge.config.ts` â€” project defaults.
- `prompts/hello.md` and `prompts/hello.test.yaml` â€” a mock-provider
  test that passes out of the box with zero API keys configured.
- Updates `.gitignore` if needed.

## 4. Add a real provider

Edit `promptforge.config.ts`:

```typescript
export default {
  providers: {
    defaultModels: [
      'anthropic/claude-haiku-4-5',
      'mock',
    ],
  },
};
```

(If you've added `promptforge` as a devDependency, wrap with
`defineConfig({...})` imported from `'promptforge'` for typed autocomplete.
Either form works at runtime â€” Zod validates the object shape.)

Set the key:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
```

Remove the `mockOutput` line from `prompts/hello.test.yaml` so the real
model gets called. (The scaffolded suite intentionally has no `providers:`
block, so it inherits `defaultModels` from the config above.) Run again.
Observe real latency, real cost.

## 5. Explore history

```bash
promptforge ui
```

Opens the dashboard at `http://127.0.0.1:3939`. Your run appears in the
runs list. Click in for the full detail view. Run a second time â€” the
compare view now has two runs to diff.
